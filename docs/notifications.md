# 通知機能について

## 概要
Service Worker を使用した通知機能を実装しました。期限前と期限切れの TODO を指定時刻に通知します。

**重要**: 通知は担当者でフィルタリングしている場合のみ送信されます。

## 通知タイミング
- **3営業日前**: 期限の3営業日前に通知
- **期限切れ**: 期限を過ぎた TODO を通知

## 通知時刻
以下の時刻にチェックして通知します：
- 9:00
- 12:00
- 17:00

## 使い方

### 1. 通知を有効にする
1. アプリを開く
2. 画面上部の「🔔 通知を有効にする」ボタンをクリック
3. ブラウザの通知許可ダイアログで「許可」を選択

### 2. 担当者でフィルタリング
1. 画面上部の担当者選択で自分の名前を選択
2. 「🔔 通知有効 (担当者名)」と表示されることを確認
3. これでフィルタリングされた担当者の TODO のみ通知されます

**注意**: 担当者フィルターが設定されていない場合は通知されません

### 3. 通知の受信
- アプリを開いている間、30分ごとに TODO をチェックします
- 指定時刻（9:00、12:00、17:00）の直後にアプリを開いていると通知が表示されます
- 通知をクリックするとアプリが開きます

## 技術仕様

### Service Worker
- ファイル: `frontend/public/sw.js`
- 登録: アプリ起動時に自動登録
- 通知チェック: メインスレッドからのメッセージを受信してチェック

### 営業日計算
- 土日を除外
- 日本の祝日を除外（holidays-jp API から取得）

### 制限事項
- **担当者フィルター必須**: 担当者でフィルタリングしている場合のみ通知が送信されます
- **ブラウザを閉じている場合**: 通知は送信されません
- **タブを開いている必要**: アプリのタブを開いている必要があります
- **対応ブラウザ**: Chrome、Firefox、Edge など Service Worker をサポートするモダンブラウザ
- **Safari**: iOS Safari では制限があります

### 通知の頻度制限
- 同じ TODO に対して重複通知を防ぐため、`tag` プロパティを使用
- 30分ごとにチェックしますが、時刻制限（9:00、12:00、17:00）により実際の通知は1日3回まで

## トラブルシューティング

### 通知が表示されない場合
1. **担当者フィルターを設定しているか確認**
   - 画面上部で担当者を選択してください
   - 「🔔 通知有効 (担当者名)」と表示されていることを確認
2. ブラウザの通知権限を確認
   - Chrome: 設定 > プライバシーとセキュリティ > サイトの設定 > 通知
3. アプリのタブが開いていることを確認
4. Service Worker が登録されているか確認
   - Chrome DevTools > Application > Service Workers

### 通知を無効にしたい場合
1. ブラウザの設定から通知を無効化
2. または、ブラウザの通知権限を拒否

## 開発者向け

### デバッグ
```javascript
// Service Worker のログを確認
// Chrome DevTools > Console で確認

// Service Worker を手動で登録解除
navigator.serviceWorker.getRegistrations().then(registrations => {
  registrations.forEach(reg => reg.unregister());
});
```

### テスト
通知のテストは時刻制限により難しいため、開発時は時刻チェックを一時的にコメントアウトしてテストできます。
